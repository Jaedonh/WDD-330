Notes

  • Chapter 10 - Errors, Exceptions, and Warnings

    • Errors are caused when something goes wrong in the code
    • Errors are usually caused by three types of things
      1. System Errors, a problem with the system or external devices
      2. Programmer Errors, program contains invalid syntax, typos, or faulty logic
      3. User Error, user has entered data in incorrectly which program cannot handle
    • Exceptions are errors that get returned values that can be used to then fix the error
    • Exceptions have stack traces which are the path of errors leading back to the root of the issue and helps to identify what went wrong
    • Warnings are errors that aren't enough to cause the program to crash but will display a message of the issue

  • Chapter 10 - The Importance of Testing and Debugging

    • Errors can happen both silently or loudly
    • A developers job is to ensure that a failure is visible so that it can be caught and corrected
    • As such testing must be done many times over to ensure that as many bugs and errors are caught as possible

  • Chapter 10 - Strict Mode

    • Strict mode is a way to run a program that produces more errors
    • It does so by even taking programming styles considered to be in poor taste and consider them errors
    • It also encourages and promotes clearer and more refined code because of the strict guidelines

  • Chapter 10 - Feature Detection

    • Because languages like JavaScript are constantly being updated and evolving there can be features that are not yet supported in certain browsers
    • To prevent an issue there is the feature detection method which detects if a method or object exists first and then if it does will continue with the assigned code block

  • Chapter 10 - Debugging in the Browser

    • Debugging is the process of finding bugs or errors and then dealing with them
    • Debugging needs to be done piece by piece to ensure all the code is properly functioning, which is where breakpoints are handy
    • Breakpoints are where the programmer chooses to halt the progress of the code to view the different values
    • There are many tools and methods to help create breakpoints, such as the alert() method, the console object to log events, and debugging tools in browsers
