Week 4 Readings

Notes

  • Chapter 8 - Forms

    • Forms are made up of form elements such as buttons, input fields, and select menus.
    • Traditionally, forms where sent to the backend to be processed. Now a days it is possible to process a form in the front end before that using JavaScript
    • Each form control has an initial value that is set but can altered as the user interacts and enter information into the form.

  • Chapter 8 - A Searching Example

    • You can access form elements by using the functions document.forms
    • A form has many values and elements that can be used or edited
    • You can do things like blur, edit form values, and access button types

  • Chapter 8 - Form Validation

    • Form Validation is the process of checking whether a user correctly entered the information
    • Validation can occur on either the client side or the server side, it is advisable to use both
    • Client side should be a script that helps inform the user about the error before it is submitted

  • Chapter 8 - Disabling the Submit Button

    • A good technique that can be employed is to disable the submit button if errors occur

  • Chapter 15 - Libraries

    • A library is a piece of code that gives several methods to make tasks easier to achieve
    • Examples of some commonly used libraries are ones like jQuery, Underscore, and the DOM API
    • The advantages to libraries are that they are thoroughly tested and have large amounts of online support
    • Some disadvantages to libraries are can increase the amount of code used, they might not work how you specifically want, and they can slow the code down as well.

  • Chapter 15 - Modular JavaScript

    • A module is a self-contained piece of code containing functions and methods to be later used
    • Modules make it so that code can be loosely coupled and changed if desired, in short it allows a lot more functionality to be implemented
    • Exporting allows for the functions and methods to be called from other places

  • Chapter 15 - MVC Frameworks

    • MVC is a design pattern for software, Model-View-Controller
    • Model, objects that allow for creating, reading, updating, and deleting pieces of info about the app
    • View, provides the visual representation of the model showing all the relevant information
    • Controller, links models and views together. It responds to events, processes the data, and updates accordingly

  • Chapter 15 - Templates

    • MVCs use templating languages to insert dynamic data into the page
    • Templates are written and then compiled into HTML
