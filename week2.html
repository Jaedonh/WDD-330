Chapter 2: Programming Basics

  • Comments

    • Use // for single line comments
    • Use /* and */ for multiline comments
    • Good practice to include comments detailing what your code does
    • Tools such as JSDoc Toolkit, Docco, and YUIDoc can make documentation from comments

  • JavaScript Grammar

    • JS Syntax is known as C-Style syntax because it's like C Programming
    • JS Program is made up of of series of statements
    • Best practice is to write pieces of code on separate lines and ended with semi-colon ;
    • A block is a series of statements inside curly brackets { }
    • Whitespace is used to make clean, readable code

  • Reserved Words

    • These are words that cannot be used for naming variables
    • Many of these words are used by the language already for various coding functions

  • Primitive Data Types

    • JavaScript has 7 data types, six of which are primitive data types
    • String: A piece of text data
    • Symbol(Only introduced in ES6 as a primitive data type): a representation of a unique identifier
    • Numbers: Positive or negative integers
    • Boolean: True of False statement derived by checking statements in JavaScript
    • Undefined: Declared variables that have not yet been assigned
    • Null: Thought of as a placehold, is it means no value. Similar to undefined but acts differently

Chapter 3: Arrays, Logics, and Loops

  • Arrays

    • An array is an ordered list of values
    • To create an array us a pair of square brackets like this, const myarray = [];

  • Sets

    • A set is a data structure representing a collection of unique values with no duplicates
    • You create a new set by using the statement, new Set()

  • Maps

    • A map is another data structure that keeps a list of key and value pairs similar to hash tables

  • Logic

    • Logic, or logical conditions allow the programmer to control the flow of a program
    • Different blocks of code depending upon logical conditions such as if-else statements

  • Loops

    • Loops are blocks of code that are repeated again and again until a certain condition is met

Chapter 4: Functions

  • Defining Functions

    • Functions are chunks of codes that act essentially as mini-programs
    • You define and create the function by using the following statement
      function hello() {
      code goes here
      }
    • You can assign functions to variables

  • Invoking a Function

    • To invoke a function means to run the code within the function
    • To invoke a function in a program you simply use the part that has the (), such as hello();

  • Return Values

    • All functions return a value
    • This value can be specified by using the return keyword

  • Parameters and Arguments

    • These are terms that are used interchangeably to represent values that are considered input for the function created
    • You can set the parameters or input for a function by using the inside of the () when defining
    • Ex: function hello(x, y){
      return x * y;
      }
    • Then when using the function you replace the x and y with your desired data, for example if you used hello(1, 2); you would get a return value of 2 * 1 which is 2
